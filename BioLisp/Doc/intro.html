The Gap Between Biologists and Biocomputation

There are a huge number of biological knowledge and data bases. Some, like TAIR, are dedicated to collecting everything about a single organism, and some, like KEGG or the NCBI, are dedicated to gathering knowledge about a broad range of organisms. Although these are generally useful and useable for the limited set of explorations that they are preprogrammed to do, few have "open functionality." Open functionality is what programming languages have: You can write any program you want (given some persistence and know-how). Open functionality is important for computational biology because biology is a living science; Biologists (and computational biologists) are always thinking up new questions that they would like answered, but which are not yet addresed by existing data and knowledge bases. Even if the data or knowledge is already easily available, it usually takes a skilled programmer, and significant effort, to implement a new computation. In part this is just the time required for engineering, but also when one is putting together your own algorithms, one has to get together all the data, reformat it to work with one's program, and install a bunch of tools, not to mention having to know some arcane programming language! As a result, nearly every lab depends upon software engineers, who are usually not biologists, and who spend huge amounts of time putting together programs to answer specific biological questions. As a result, although biologists are swimming in data, we usually need programmers in order to ask novel questions of it.

BioBike: A Tool for Biological Thought.

BioBike is an approach to reducing the distance between biologists and biocomputation by providing an integrated, on-line, programmable biological knowledge base. In many other scientific areas, students use programmable tools to address complex questions. For example, most undergraduate engineers are trained to use matlab, a matrix-based programming language, as a tool for thought in engineering. But biologists don't have a tool of that sort, and certainly nothing like that is integrated into their training. BioBike is a sort of matlab for biological knowledge, where instead of matrices you have a knowledge base which integrates biological knowledge about particular organisms, their genomes and proteomes, and which provides tools to conveniently operate on that knowledge.

BioBike itself is a very simple, very efficient biology-specific programming language embedded in an integrated biological knowledge-base, and accessible through the web. In the best case, biologists themselves would be able to log into the system and write their own simple programs to ask novel biological questions without the help of (or with very little help from) software engineers.

This Multi-Cyano BioBike Web Listener

The CIW/NASA Multi-Cyano BioBike server is a first try at implementing the complete BioBike vision. The knowledge base will eventually contain integrated knowledge about all of the fully sequenced cyanobacterial strains. (Currently we have 13 in the set, and expect to include perhaps 20 in the next year as their sequences are published.) The BioBike Web Listener (hosted at The Carnegie Inst. of Washington) currently implements a nearly complete core of the BioBike language, and has on-board an extensive array of tools for both sequence and microarray data analysis. The knowledge base for each cyanobacterium is linked into a Gene Ontology backbone, and the knowledge base includes the precomputed "cross blast" and orthology of every gene against every other gene in every other organism.

Most importantly, the core BioBike language itself -- including a very efficient compiler to make complex analyses run fast -- is completely implemented, and available for immediate use. Cyanobacteriologists can log onto the server TODAY and begin to ask biological questions about these organisms, including importing and integrating their own data into the system.

The BioBike Vision and the Knowledge Operating System.

We want to make BioBike as simple and flexible a tool for working with biological knowledge as Excel is for working with tables. Even beyond this, we are trying to make BioBike into a complete Knowledge Operating System; Whereas Unix and Windows are operating systems for ASCII strings and files, and there are operating systems for tables (such as Oracle), we envision BioBike, and its more general spinoffs, as operating systems for knowledge! The current BioBike implementation actually has many of the facilities of an operating system, including an integrated file system and multi-processing, but the representational basis for BioBike is knowledge, not files or tables. To reach the vision of an operating system for knowledge will take more work than our small, mostly volunteeer team has in us, but it's something to strive for.

How to try out BioBike, and help develop The BioBike Vision

The BioBike Vision is of biologist-driven open functionality ranging over integrated knowledge and data bases. The CIW/NASA Multi-Cyano server is just the first experimental implementation of this vision. As we interact with more and more biologists and computational biologists, we expect that the vision will grow and evolve.

You can experiment with the demo version of the CIW/NASA Multi-Cyano BioBike server by accessing the BioBike documentation core, and clicking through to the demo server. If you'd like to help us reach the vision, please let Jeff Shrager (jshrager@stanford.edu), the project director.
