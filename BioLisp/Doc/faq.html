Q: What is BioBike?

The BioBike system is a biology-specific programming language embedded in an integrated biological knowledge-base, all accessible through the web. In the best case, biologists themselves would be able to log into the system and write their own programs to ask novel biological questions without the help of (or with very little help from) software engineers.

More: The BioBike Vision

Q: What's the point of BioBike?

BioBike is an approach to reducing the distance between biologists and biocomputation by providing an integrated, on-line, programmable biological knowledge base. In many other scientific areas students use programmable tools to address complex questions. For example, most undergraduate engineers are trained to use matlab, a matrix-based programming language. But biologists don't have a tool of that sort, and certainly nothing like that is integrated into their training. BioBike is a sort of matlab for biological knowledge, where instead of matrices you have a knowledge base which integrates biological knowledge about particular organisms, their genomes and proteomes, and which provides tools to conveniently operate on that knowledge.
More: The BioBike Vision

Q: Why is it called BioBike?

It used to be called "BioLingua" but someone who was using that name forced us to change it. Bikes are the most efficient means of transportation known, and there are various possible acronyms. ("Big Integrated Knowledge Environment" is a favorite.)

Q: Who invented BioBike?

BioBike was envisioned by Jeff Shrager and the core code was written by JP Massar. Many people subsequently contributed to it, most significantly: Mike Travers (who built the frame system and many of the knowledge access components), and Jeff Elhai.

Q: How does BioBike go beyond typical biological knowledegbases?

Online bioloigical data or knowledge bases (of which there are zillions!) are very useful for simple lookup sorts of work, but they typically operate in a "vending machine" model: You put in a query, and you get out an answer. If you're lucky, you can do some pre-packaged computations in the database, and maybe even do them on multiple results from a query. If that's all you need to do, great!

BioBike is a full fledged efficent programming language embedded in an integrated knowledge base -- all totally web-accessible. This is a completely different animal than the vending-machine sort of knowledge bases you're probably used to. Instead of using web commands you use programming language constructs, and you can pass the results of one computation on to the next (like in any sort of programming languages), which allows you to combine operations. Moreover, BioBike has built-in all of the typical bioinformatic tools (Blast, Clustal, etc.) so that not only can you do your own computations in the database, but you can call upon these other tools -- all directly through the BioBike interface!

Q: Is it hard to use?

It's not trivial, like a vending machine database would be, but it's much more powerful, and we think that the power/effort ratio is worth it. You have to learn a little programming -- usually very little -- to get simple things done. The main BioBike language is called BioLisp -- it's just common lisp with a bunch of biological functionality added in. But you don't have learn Lisp (although we think you should! :-)

There are several different specialized languages developed for BioBike that are there to help biologists get the most out of the system. The main one, called BBL, was developed by Jeff Elhai at Virginia Commonwealth University. BBL also has a visual programming language, called vBBL. You can see a walk-through tour of vBBL here.

Q: Why Lisp?

Because it's the right language for the job!

Q: What about those specialized Bio* languages (BioPerl, etc)?

BioPerl, BioJava, BioPython, etc. certainly have their place, but all they offer is a programming language with some biological functionality added in. In order to use them you have to install them, and then install databases, and then figure out how to put them all together. In BioBike we've done all that work for you, and you don't even have to have your own computer (except to run a web browser)!

Q: Who paid for it?

Many people in various ways. NASA gave us seed funding, Franz Inc and Lispworks, Inc. loaned us Lisp licenses, most of the initial engineering was done on a volunteer basis, and much continues that way. NSF now supports some of the BioBike development, but much is still done on a volunteer basis.

Q: What lisps does BioBike run on?

Theoretically any "compliant" implementation of Common Lisp, but we do most of our testing under Franz ACL 8.0 and LispWorks. The BioBike servers are running Franz ACL on Linux.

Q: What's the best way to get started learning BioBike?

Note: The demo server is no longer supported. Click here to connect to the new BioBike server page.

First play with the BioBike demo server. There are numerous "live tutorials" that you can get to by logging into the demo server, and then using the pulldown at the bottom to go to: Help ... > Live Tutorials. This will give you a list of Live Tutorials. We suggest starting the the first one, called "A Tour of BioBike".

Q: Is there a BioBike course?

The online tutorials (esp. the Live ones) are a good start, but they aren't really a lesson plan. Jeff Elhai (of Virgnia Commonwealth University) and Jeff Shrager (of The Carnegie Inst. and Stanford) both teach BioBike courses and have a lot of course materials, but most of it eventually goes online, either in the BioBike static docs, or in the Live Tutorials.

Q: What if I want to get my own BioBike server at my own lab?

First, think twice about whether you really need this? Can you use our demo server or the Cyano sever for your work? If we don't have your favorite genome on a server, ask us (via the BioBike Google Group) and we may be able to put it up for you on either the demo server, or one a private server. If you really really want to try to bring up your own BioBike server, you have to get a Lisp platform (see below), and then download the sources and install it. There is documentation in the core system documentation to help you do this.

It takes some effort to setup and support a BioBike server -- esp. with complex systems of biological data, knowledge, and tools. You need to be knowledgeable about Lisp, Linux, and biocomputing tools. This is a big part of the reason that we developed BioBike as a client-server system in the first place; we had in mind that a whole community of interest in a particular organism or class of organisms would buy one server and one Lisp license and pay for one person's time to support it for the whole community. In our case it was the cyanobacterial community, which is why the main BioBike server contains cyanobacterial models.

The community might even put in a small grant proposal to build themselves a BioBike server, or add it on as a small cost to their already probably pretty big genome database grants. Those costs and effort are then amortized over the whole user community. And more importantly, someone else gets to do the dirty work of integrating the data and supporting the server. When every lab has to do their own software setup it can be painful and expensive, but BioBike lets you amortize the cost and effort over a whole community of users.

If you're interested in putting together a proposal to develop a BioBike server for your community, Jeff Shrager would be more than happy to help you project your costs.

Q: How do I get a Lisp that can run BioBike?

The best way is to buy it from Franz, Inc. If you're an academic or non-profit, we can usually get you a good deal on a BioBike specific license. Contact Jeff Shrager for more information on this. The cost is currently something like $1250 for the first year, and ~1/2 of that for each continuing year of the license.
subQ: Isn't a thousand bucks a little steep for a programming language platform? Why didn't you use a free Lisp?
First off, it's only a thousand dollars the first year, and it's not unreasonable compared to how much you pay for computers, HPLC machines, lab materials, and books. We chose to use a commercial Lisp engine because it is well supported and does what we need it to do. No freeware Lisp platforms are as well supported as the for-profit Lisps, and none do what we need them to do in all situations. (And when ACL doesn't do what we need it to do, we complain and they fix it!)

However, theoretically any compliant implementation of Common Lisp will run the BioBike basics, but possibly not some of the detailed multi-tasking code, which is ACL-dependent. (There's no CL standard for these sorts of things.) If you want to try to port BioBike to a free Lisp, we'll work with you to do that. Someone actully ported most of BioBike to SBCL once, and there's still some conditional compilation code left in the system. We're happy for you to try to do this, and we'll help you as much as we can. Also, we'll be happy to patch back your work into the code as conditional compiles.
