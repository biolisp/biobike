The main help command is, unsurprisingly, called "help" ! 

  >> (help)
or just plain 
  >> help
and it gets you 
Email support: biobike@lists.stanford.edu
Use the 'Documentation' link at the bottom of the page for more help,
  site search capabilities and tutorials.
HELP usage:
  (help "a string") -- find functions related to the string
  (help a-symbol) -- print documentation for the symbol
For detailed information on help itself, type: (help help)
and click on the 'HELP' link to see the full HELP documentation.

If you type (help help) you'll get: 

HELP	(&OPTIONAL THING &REST ARGS)
  Usage: (help THING ARGS) If THING names a Biobike primitive exactly, the ...
With the word HELP hyperlinked. If you click on it you'll see the entire description of the Biobike HELP function, along with the source code that implements it.

You get a list of functions and variables in the Biobike system which have something to do with 'ORGANISM', and for each such symbol, the function's argument list (or the variable's value) and the initial part of its documentation description. Each symbol so listed is also hyperlinked and if you click on the link you'll get a complete description of the function, along with source code, or more information about the variable. You also gets hints on how to use the HELP command to limit or expand the number of entries displayed.

Now let's see the help on a specific function, load-organism: 

  >> (help load-organism)
::
LOAD-ORGANISM	(ORGANISM &KEY RELOAD? ORGANISM-INFO? GE...)
  Load information from the database about an organism into memory. 
Creates genome frames for each contig of the organism, and possibly 
gene, transcript and protein frames if these tables exist in 
the database.  Modifies *loaded-organisms* appropriately.
By querying for a specific function you get the entire documentation description, as well as its argument list and a link to display to you the source code for the function. From that display, you can look at the entire file the function is defined it, or get a directory listing of all the files in the same directory.

As you see, we find functions and variables with either "load" or "organism" (or both!) in them, and another entry, MEME, which the matching algorithm thinks might be related to what you are asking for.

There are a number of other important ways to get help in Biobike just using commands. HELP-LISP is a function which will print out Common Lisp symbols that match your input, and there are the Common Lisp functions APROPOS, APROPOS-LIST, and DESCRIBE.

APROPOS is a lot like HELP-LISP, while DESCRIBE will print out a description of any object you give it: 

  >> (describe :fred)
:: 
:FRED is a NEW SYMBOL.
  Its value is :FRED
  It is globally declared to be a constant.
  It is EXTERNAL in the KEYWORD package.

  >> (describe 1000000000000000)
:: 
1000000000000000 is a NEW BIGNUM.
 It is 50 bits wide
You can read more about them in the Common Lisp Hyperspec. Here's the entry for APROPOS, and here's the one for DESCRIBE

Now, we've mentioned that the names of the functions in the foregoing are all marked as hyperlinks. That is, you can click through them to something. I wonder what it could be?

Let's try clicking through a function. If you have access to the Biobike system, click on LOAD-ORGANISM. All sort of interesting things appear:

  • The argument list
  • The name of the source file containing LOAD-ORGANISM, clickable
  • The name of the source directory, clickable
  • The documentation for LOAD-ORGANISM
  • The source code listing

But that's not all! The source code listing itself contains clickable items, which are the names of functions and macros that LOAD-ORGANISM calls. Some of these are standard Lisp functions (in purple), and some of them are Biobike primitives (in green)

You can look at the source file by clicking on its name; the same purple and green links will appear along with the complete file.

You can look at a simple listing of the source directory, and from there examine other source files in the directory or other higher-level or sub-directories.

Besides HELP, another means of viewing what Biobike primitives are available is through the 'Bio Primitives' link near the bottom of the Listener.

This will give you a list of categories, which you can click into ('+' or '*') to get a listing of primitives in that category. Or you can click on the 'alphabetical listing' link near the top of the page to get a listing of all the Biobike primitives.

Finally, you can always email the Biobike support team with your questions if you get stuck.
